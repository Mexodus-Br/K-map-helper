# 逻辑表达式卡诺图与化简程序

这是一个用 C++ 编写的逻辑表达式化简工具，可用于：

- 读取形如 `Z(a,b,c,...)=...` 的逻辑表达式
- 输出对应的卡诺图
- 输出最简与或表达式 `SOP`
- 输出最简或与表达式 `POS`
- 输出最小项表达式，例如 `Σm(1,3,5,7)`
- 输出最大项表达式，例如 `ΠM(0,2,4,6)`

程序最多支持 25 个变量，变量名限定为 `a-y`。

## 功能特点

- 支持小写字母表示原变量，大写字母表示反变量
  - 例如 `aB` 表示 `a & ~b`
- 支持前缀非运算符 `!` 和 `~`
  - 推荐写法：`!`
  - `!` 与 `~` 完全等价
  - 例如 `!a = ~a = A`
- 支持一般布尔表达式，不只限于“积项相加”
- 支持如下运算写法
  - `+` 表示或
  - `*` 或 `.` 表示与
  - 相邻项默认也表示与
  - 括号 `()` 改变优先级
- 支持交互式循环输入
- Windows 终端下会主动设置控制台编码，减少中文乱码问题
- 支持通过 WebAssembly 在浏览器中调用同一套化简核心
- 提供开箱即用的网页前端 `[web/index.html](web/index.html)`
- 最小项/最大项编号按输入变量表的自然二进制顺序生成
  - 例如 `Z(a,b,c)` 中，`abc=000,001,010,...,111` 对应编号 `0,1,2,...,7`

## 输入格式

输入必须以函数头开头：

```text
Z(a,b,c,...)=表达式
```

其中：

- 函数名目前只要求非空，程序不会限制它必须是字母、数字或下划线
- 左侧变量列表必须是互不重复的 `a-y` 小写字母
- 右侧表达式中：
  - 小写字母表示原变量
  - 大写字母表示反变量
  - `!` 与 `~` 表示前缀非，二者等价，推荐使用 `!`

## 支持的表达式示例

```text
Z(a,b,c)=aB+cdE
Z(a,b,c)=a*(B+c).c
Z(a,b,c)=!a+bc
Z(a,b,c)=a!b+!(b+c)
Z(a,b,c)=~(a+b)c
Z(a,b,c,d)=((a+b)(C+d))+aB(c+d)
Z(a,b,c,d,e)=!(d+e)(a+b+c)(B+C+D+E)(a+B+c+D+e)
```

说明：

- `!a`、`~a`、`A` 等价
- `a!b` 会按 `a & !b` 解析
- `!(ab)` 或 `~(ab)` 表示对整个乘积取反

## 编译方法

在项目目录下执行：

```powershell
g++ -std=c++17 -O2 -Wall -Wextra -pedantic main.cpp -o logic_kmap.exe
```

## 运行方法

```powershell
.\logic_kmap.exe
```

程序启动后会显示输入提示。输入一条表达式并回车后，会输出：

- 卡诺图
- 最简与或表达式 `SOP`
- 最简或与表达式 `POS`
- 最小项表达式 `Σm(...)`
- 最大项表达式 `ΠM(...)`

输入空行，或输入 `exit`、`quit`、`q` 可退出程序。

## WebAssembly 构建与网页运行

如果你希望在浏览器中使用本程序：

1. 安装并激活 Emscripten，确保 `emcc` 已加入 `PATH`
2. 在项目目录执行：

```bash
./build_wasm.sh
```

执行成功后会生成：

- `web/kmap.js`
- `web/kmap.wasm`

然后直接打开 `web/index.html` 即可在页面中输入表达式并查看：

- 最简与或表达式 `SOP`
- 最简或与表达式 `POS`
- 最小项表达式 `Σm(...)`
- 最大项表达式 `ΠM(...)`
- 卡诺图文本与网格输出

注意：

- 如果修改了 `main.cpp` 的 Web 解析逻辑或导出接口，需要重新执行 `build_wasm.sh`
- 否则页面可能会加载旧版 `web/kmap.js` / `web/kmap.wasm`

## 运行示例

输入：

```text
Z(a,b,c,d,e)=!(d+e)(a+b+c)(B+C+D+E)(a+B+c+D+e)
```

输出内容包括：

- 对应卡诺图
- 最简与或表达式
- 最简或与表达式
- 最小项表达式
- 最大项表达式

## 卡诺图说明

- 变量表中靠前的一半变量作为行变量，靠后的一半变量作为列变量
- 行列都采用 Gray 顺序输出
- 当变量过多导致卡诺图尺寸过大时，程序只输出卡诺图维度信息，不展开完整表格

## 已知说明

- 程序内部使用 BDD 表示逻辑函数，再从函数覆盖中生成 `SOP/POS`
- 最小项表达式和最大项表达式需要枚举真值编号；当项数过多时，程序会提示“项数过多，未能在限制内展开”
- 对于规模很大的函数，`SOP` 或 `POS` 的覆盖展开可能过大，此时程序会提示“路径数过多，未能在限制内展开”
- 变量上限是 25，但变量越多，卡诺图越不适合完整打印

## 文件说明

- `main.cpp`：主程序源码
- `build_wasm.sh`：WebAssembly 构建脚本
- `web/index.html`：网页前端入口
- `web/kmap.js` / `web/kmap.wasm`：WASM 构建输出
